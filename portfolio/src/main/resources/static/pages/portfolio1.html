Question Answering System

Introduction Web-based Question and answering system with Natural language process (NLP) that can be handed in Humen questions in order to provide a proper answer.

Building Question answering (QA) system 은 Natural language processing (NLP) 기반의 사용자 질의문장 이해 그리고 답변하는 시스템입니다. 일반적으로 이 시스템은 두 가지 타입 사용자 질의문장을 이해합니다. 첫번째는 Fact-based Question 타입의 문장을 이해합니다. 예를 들면 "Where is the capital of South Korea?" 라는 식의 정확한 답이 있는 질문 타입을 말합니다. 답변을 위해서는 Online 자원을 사용합니다. 두번째로는 Tuffy-based Question 타입의 문장을 이해합니다. Tuffy란 "Markov Logic Network (MLN)" 규칙 기반의 Java Library를 말합니다. Tuffy는 사용자의 질문을 저장하고 연상, 학습하여 답을 유추하는데 도움을 줍니다. 예를 들면 "Anna is a smoker. Will Anna have cancer?" 라는 식의 질문을 했을 경우 시스템 내의 데이터 베이스에서 "Smoker" 와 "Cancer"의 연관성을 추적한 뒤 답변을 사용자에게 제공합니다. 나아가 사용자가 현재 아픈 몸상태에 대해서 질문을 했을 경우, 질문의 Keywords를 현존하는 질병과 연관하여 현재 어떠한 질병을 가지고 있는 지 유추 가능합니다. 하지만 보다 많은 답변을 위해서는 많은 데이터 베이스가 필요로 하며 많은 양의 데이터베이스를 처리하기 위해서는 High performance computing system을 요구해야 하는 단점이 있습니다. 현재 이 프로젝트는 Java와 Spring Framework 기반으로 제작되어 Laptop, Mobile 등으로 접속 가능 하며 Fact-based answering module 은 Google, DBpedia, Wiki 등 웹 Searchers 확장을 통해서 보다 풍부한 Answering 이 가능하며 Tuffy-based answering module 은 서버내에 학습되어 있는 데이터 베이스만큼의 Answering 가능합니다.
Technique Spring Boot, Bootstrap and OpenEphyra framework, PostgreSQL, NLP, MLN.
